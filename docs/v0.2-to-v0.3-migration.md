# v0.2 → v0.3 migration

## What changed

The ontology IRI shape changed from hash semantics to slash semantics:

| | v0.2 | v0.3 |
|---|---|---|
| Namespace | `https://w3id.org/cdisc/usdm/v4#` | `https://w3id.org/cdisc/usdm/v4/` |
| Class IRI example | `…/v4#Activity` | `…/v4/Activity` |
| Property IRI example | `…/v4#Activity-name` | `…/v4/Activity-name` |
| Ontology resource | `<…/v4>` | `<…/v4/>` |

This is a breaking change to every minted IRI in the ontology. Triple
counts and structural counts are unchanged: the v0.3 TTL contains the
same 8,173 triples, 86 classes, 693 properties, and 57 codelist
bindings as v0.2.

## Why it changed

The driver was per-class HTML rendering for human consumers landing on
a class or property IRI. Slash semantics gives each IRI its own
server-side dereferenceable form, allowing `…/v4/Activity` to redirect
to a per-class HTML page (and `…/v4/Activity-name` to the same page
anchored at the property fragment). Hash semantics forces every IRI to
share a single document, which combines poorly with HTML renderers
(pyLODE, WIDOCO) that derive anchor slugs from `skos:prefLabel` rather
than the IRI fragment. Full rationale in
[iri-and-governance.md](iri-and-governance.md).

## Acceptable cost

The breaking change is acceptable because v0.2 was four days old when
v0.3 was scoped, with no known external consumers and no external
citations of the v0.2 IRIs. There is no equivalent window for a future
namespace flip — minted IRIs in v0.3 are treated as stable from v0.3
forward.

## For consumers of v0.2

If a tool or document references a v0.2 IRI, the mechanical
substitution is `s|v4#|v4/|g` (replace one `#` with one `/`). The v0.2
TTL is recoverable at the `v0.2.0` git tag; the source SHA-256 is
recorded in the v0.2 commit history.
